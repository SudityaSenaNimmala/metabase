1. Overview

The Metabase Dashboard Auto-Clone Tool is a web-based application designed for migration managers to automate the creation and management of customer-specific Metabase dashboards. Instead of manually creating dashboards for each customer migration, this tool automatically clones pre-configured source dashboards and maintains them throughout the migration lifecycle.

üí° Quick Summary: This tool saves hours of manual work by automatically creating and managing Metabase dashboards for each CloudFuze customer migration, ensuring consistent reporting across all migration projects.

2. What This Tool Does

Primary Function

The tool monitors your MongoDB database for new customer migrations and automatically clones the appropriate Metabase dashboards for each customer based on their migration type (Content, Message, or Email).

How It Works

  1. Auto-Detection: The tool continuously checks MongoDB for new customers who need dashboards
  2. Automatic Cloning: When a new customer is detected, it automatically clones the appropriate source dashboard
  3. Organization: Creates a dedicated collection in Metabase for each customer
  4. URL Assignment: Generates and stores unique Metabase URLs for easy access
  5. Activity Tracking: Logs all operations for audit and troubleshooting purposes

5. Dashboard Types

The tool supports three types of customer migrations, each with its own color coding:

CONTENT Content Dashboards

  • Purpose: Track customers migrating content and files
  • Metrics: Files migrated, folders processed, transfer rates, storage utilization
  • Color: Green indicator

MESSAGE Message Dashboards

  • Purpose: Monitor customers migrating messaging data
  • Metrics: Message counts, conversation threads, user activities, channel migrations
  • Color: Purple indicator

EMAIL Email Dashboards

  • Purpose: Track customers migrating email data
  • Metrics: Mailbox migrations, email counts, attachment processing, folder structures
  • Color: Orange indicator
üí° Tip: Color indicators help you quickly identify dashboard types at a glance. This is especially useful when managing multiple customer migrations simultaneously.

6. Using the Tool

Home Dashboard View

When you open the tool, you'll see:

  • Header: Your profile picture and navigation options
  • Dashboard Count Cards: Visual summary showing total dashboards by type
  • Manual Run Button: "Run Now" button to trigger immediate dashboard creation
  • Dashboard List: Grid view of all existing customer dashboards

Performing a Manual Run

To manually trigger dashboard creation:

  1. Click the "Run Now" button in the top section
  2. The tool will:
    • Check MongoDB for customers without dashboards
    • Create missing dashboards automatically
    • Show progress notifications for each creation
  3. Wait for the completion notification
  4. New dashboards will appear in the list automatically
‚ö†Ô∏è When to Use Manual Run:
  • Immediately after adding a new customer to MongoDB
  • After updating source dashboard templates
  • When scheduled run frequency is not sufficient
  • For urgent customer onboarding requirements

Understanding Dashboard Cards

Each dashboard card displays:

  • Dashboard Name: Customer name with dashboard type
  • Description: "No description" (can be updated in Metabase)
  • ID Number: Unique Metabase dashboard identifier
  • Type Badge: Color-coded badge (CONTENT/MESSAGE/EMAIL)
  • Action Icons: Edit, Delete, and Update icons
  • Open Button: Blue button to view dashboard in Metabase

7. Dashboard Management

Viewing a Dashboard

Steps:

  1. Locate the customer dashboard in the list
  2. Click the "Open" button (blue button at bottom)
  3. Dashboard opens in a new tab in Metabase
  4. Interact with the dashboard as needed (filters, drill-throughs, exports)

Updating a Dashboard

Use this feature when source templates have been updated and you want to refresh a customer's dashboard with the latest changes.

Steps:

  1. Click the Update icon (üîÑ circular arrows) on the dashboard card
  2. The card flips to show a confirmation message explaining the update process
  3. Review the message and click "Proceed" to continue or "Cancel" to abort
  4. If you proceed:
    • Card flips to show a progress view with progress bar
    • Progress bar updates in real-time showing completion percentage
    • Status messages show current step (e.g., "Cloning dashboard", "Creating questions")
    • Estimated time: 2-5 minutes depending on dashboard complexity
  5. Optional: Click "Cancel Update" button anytime to abort the operation
  6. On completion:
    • Card automatically flips back to normal view
    • Dashboard list refreshes with updated data
    • Activity log records the update with your name
üîí Safety Features:
  • The update process is safe - the old dashboard remains intact until the new one is successfully cloned
  • If update fails or is cancelled, the original dashboard stays active
  • No data loss risk during update operations
  • All linked dashboards and questions are updated together

Deleting a Dashboard

Use this feature to permanently remove a customer's dashboard after migration completion or project cancellation.

Steps:

  1. Click the Delete icon (üóëÔ∏è trash bin) on the dashboard card
  2. Card flips to show delete confirmation with warning message
  3. Read the warning: "This action cannot be undone"
  4. Click "Delete" to proceed or "Cancel" to abort
  5. If confirmed:
    • Dashboard is removed from Metabase immediately
    • All associated questions/cards are deleted
    • Customer collection is removed
    • Dashboard disappears from the list
    • Activity log records the deletion with your name
‚ö†Ô∏è What Gets Deleted:
  • The main customer dashboard
  • All linked/drill-through dashboards
  • All questions/cards in the customer collection
  • The customer collection itself
  • All references in the tool's database
‚ö†Ô∏è Before Deleting:
  • Confirm the migration is complete
  • Export any needed data from Metabase for records
  • Notify relevant stakeholders
  • Verify you're deleting the correct dashboard

Renaming a Dashboard

Steps:

  1. Click the Edit icon (‚úèÔ∏è pencil) on the dashboard card

  4. Type the new name (or edit the existing name)
  5. Click "Save" to update or "Cancel" to abort
  6. If saved:

9. Merge Dashboards

What is Dashboard Merging?

The merge feature allows you to combine multiple customer dashboards into a single unified dashboard. This is useful when a customer has multiple migration types running simultaneously.

When to Use Merge

  • Customer has multiple migration types (e.g., both Content and Email)
  • Need a consolidated view across all customer data
  • Want to create custom dashboard combinations for reporting
  • Executive dashboard showing complete migration status

How to Merge Dashboards

Steps:

  1. Click the "Merge Dashboards" button in the navigation
  2. Select Dashboards:
    • Click the "+" icon in the top-right corner of each dashboard you want to merge
    • Selected dashboards show a blue checkmark
    • Click the "+" icon again to deselect if needed
    • Must select at least 2 dashboards to proceed
  3. Enter Merge Name:
    • Type a descriptive name in the toolbar input field
    • Example: "Contoso - Complete Migration View"
    • Example: "Q1 2024 - All Migrations Summary"
  4. Click "Merge Selected" button in the toolbar
  5. Wait for merge completion (usually 30-60 seconds)
  6. Success notification appears when complete
  7. New merged dashboard appears in the "Merged" tab

Managing Merged Dashboards

Merged dashboards have their own dedicated section:

  1. Click the "Merged" tab in the navigation
  2. View all your merged dashboards
  3. Each merged card shows:
    • Merged dashboard name
    • List of source dashboards that were combined
    • Creation date
    • Your standard action buttons

Available Actions on Merged Dashboards:

  • Open: View the merged dashboard in Metabase
  • Rename: Change the merged dashboard name
  • Delete: Remove the merged dashboard
üîí Important: Deleting a merged dashboard does NOT affect the source dashboards. Source dashboards remain intact and can be merged again if needed.

10. Troubleshooting

Dashboard Not Appearing After Manual Run

Possible Causes:

  • Customer data not properly configured in MongoDB
  • Customer already has a dashboard assigned
  • Source dashboard not configured for that migration type
  • Database connection issue

Solutions:

  1. Check MongoDB to verify customer entry exists
  2. Verify customer has dashboard_id field set to null (or not set)
  3. Check Activity Log for error messages
  4. Contact CloudFuze IT support to verify source dashboard configuration
  5. Wait 2-3 minutes and refresh the page

Update Operation Failed

Possible Causes:

  • Network connectivity issues
  • Metabase API timeout (dashboard too complex)
  • Source dashboard was deleted or modified
  • Insufficient Metabase permissions

Solutions:

  1. Check Activity Log for specific error details
  2. Wait 5 minutes and try the update again
  3. Verify source dashboard still exists in Metabase
  4. Check network connection stability
  5. Contact CloudFuze IT support if error persists after retry

Cannot Open Dashboard (404 Error)

Possible Causes:

  • Dashboard was manually deleted from Metabase
  • URL is outdated or corrupted
  • Metabase permissions changed
  • Metabase server is down

Solutions:

  1. Try updating the dashboard to regenerate the URL
  2. Check if dashboard exists in Metabase manually
  3. Contact CloudFuze IT support to verify Metabase access permissions
  4. Check if Metabase is accessible (try opening Metabase directly)

Progress Bar Stuck During Update

Possible Causes:

  • Very large dashboard taking longer than expected
  • Backend process crashed
  • Network timeout
  • Metabase API not responding

Solutions:

  1. Wait 5-10 minutes - some complex dashboards take time
  2. Click "Cancel Update" button to abort the operation
  3. Refresh the browser page
  4. Check Activity Log to see if operation completed
  5. Try the operation again
  6. Contact CloudFuze IT support if problem persists

Authentication Issues

Possible Causes:

  • Session expired (logged in too long)
  • Azure AD configuration changed
  • Network connectivity issues
  • Browser cookies disabled

Solutions:

  1. Refresh the page and try again
  2. Clear browser cache and cookies
  3. Try logging out and logging in again
  4. Try a different browser (Chrome, Edge, Firefox)
  5. Contact CloudFuze IT support if problem persists

11. Best Practices

For Daily Operations

‚úÖ Do's:
  • Check Activity Log daily for any failed operations
  • Monitor auto-clone successes to ensure system is running
  • Review dashboard list weekly to identify completed migrations
  • Keep dashboard names current with customer information
  • Use manual run after bulk customer imports to MongoDB
‚ùå Don'ts:
  • Don't delete dashboards for active migrations
  • Don't update dashboards during business hours when customers are viewing them
  • Don't rename dashboards without notifying the team
  • Don't ignore failed operations in activity log
  • Don't perform bulk operations without admin approval

For Dashboard Management

Before Updating:

  • Verify the update is necessary (has source template changed?)
  • Check if anyone is currently using the dashboard
  • Schedule updates during low-usage times (evenings, weekends)
  • Test updates on non-critical dashboards first

Before Deleting:

  • Confirm migration is 100% complete
  • Export any needed data from Metabase for records
  • Notify relevant stakeholders (account manager, project team)
  • Document the deletion in your project records

Naming Conventions:

  • Use format: "[Customer Name] - [Type] Dashboard"
  • Example: "Contoso Corporation - Content Dashboard"
  • For merged: "[Customer Name] - Combined Migration View"
  • Keep names under 50 characters when possible

For Troubleshooting

  1. Always Check Activity Logs First
    • 90% of issues have logged error messages
    • Helps identify patterns in failures
    • Shows exactly when problem occurred
  2. Document Recurring Issues
    • Note exact steps that led to problem
    • Take screenshots if helpful
    • Share findings with team and admin
  3. Test on Non-Critical Dashboards
    • Use test customer dashboards when available
    • Verify results before applying to production

For Team Collaboration

  • Communication: Inform CloudFuze migration team before making bulk changes
  • Documentation: Keep notes on any custom dashboard modifications
  • Consistency: Follow CloudFuze naming and organization standards
  • Transparency: Activity logs track all actions - be accountable
